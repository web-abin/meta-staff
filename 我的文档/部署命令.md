# Hermes-Agent 部署命令（OpenCloudOS 9 / x86_64）

> 从上到下一条一条贴，跑完上一条再贴下一条。出错就把报错发给我。

```bash
# 1. 卸掉系统自带的 podman / runc / 老 docker（避免冲突）
sudo dnf remove -y docker docker-client docker-client-latest docker-common docker-latest docker-latest-logrotate docker-logrotate docker-engine podman runc buildah 2>/dev/null
```

```bash
# 2. 装基础依赖
sudo dnf install -y dnf-plugins-core git curl vim
```

```bash
# 3. 写阿里云 docker-ce 仓库
sudo tee /etc/yum.repos.d/docker-ce.repo > /dev/null <<'EOF'
[docker-ce-stable]
name=Docker CE Stable
baseurl=https://mirrors.aliyun.com/docker-ce/linux/centos/9/x86_64/stable
enabled=1
gpgcheck=1
gpgkey=https://mirrors.aliyun.com/docker-ce/linux/centos/gpg
EOF
```

```bash
# 4. 装 docker + compose
sudo dnf install -y --allowerasing docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
```

```bash
# 5. 启动 docker + 开机自启
sudo systemctl enable --now docker
```

```bash
# 6. 验证 docker 装好了（应该输出 Client/Server 版本号）
sudo docker version && sudo docker compose version
```

```bash
# 7. 当前用户加入 docker 组（免 sudo）
sudo usermod -aG docker $USER
```

```bash
# 8. ⚠️ 重要：退出 ssh 重新登录一次（让 docker 组生效）
exit
# 然后重新 ssh 登录服务器，继续下一步
```

```bash
# 9. 配 docker 镜像加速（国内拉镜像必加）
sudo mkdir -p /etc/docker
sudo tee /etc/docker/daemon.json > /dev/null <<'EOF'
{
  "registry-mirrors": [
    "https://docker.1ms.run",
    "https://docker.xuanyuan.me",
    "https://hub.rat.dev"
  ]
}
EOF
sudo systemctl restart docker
```

```bash
# 10. 验证镜像加速生效（应该看到上面三个 URL）
docker info | grep -A 3 "Registry Mirrors"
```

```bash
# 11. clone hermes-agent
git clone https://github.com/NousResearch/hermes-agent.git ~/hermes-agent
```

```bash
# 12. 进入仓库
cd ~/hermes-agent
```

```bash
# 13. 拷贝 .env 模板
cp .env.example .env
```

```bash
# 14. 生成 API key（复制下面输出的那串 hex，meta-staff 那边要用同一个）
openssl rand -hex 32
```

```bash
# 15. 编辑 .env，把下面三个变量改成对应值
vim .env
# API_SERVER_HOST=0.0.0.0
# API_SERVER_KEY=<上一步生成的 hex 字符串>
# OPENROUTER_API_KEY=sk-or-...   (或者 ANTHROPIC_API_KEY=sk-ant-...)
```

```bash
# 16. 拉镜像 + 后台起服务
docker compose up -d
```

```bash
# 17. 看启动日志，确认 api_server 监听 0.0.0.0:8642（Ctrl+C 退日志，容器仍在跑）
docker compose logs -f gateway
```

```bash
# 18. 本机健康检查
curl http://127.0.0.1:8642/health
```

```bash
# 19. 带鉴权调一次 chat completions 验证（<KEY> 换成第 14 步生成的 hex）
curl -sS http://127.0.0.1:8642/v1/chat/completions \
  -H "Authorization: Bearer <KEY>" \
  -H "Content-Type: application/json" \
  -d '{"model":"hermes-agent","messages":[{"role":"user","content":"say hi"}]}'
```

```bash
# 20. 开放系统防火墙 8642 端口
sudo firewall-cmd --permanent --add-port=8642/tcp && sudo firewall-cmd --reload
```

```bash
# 21. ⚠️ 云控制台安全组也要开 8642/tcp（这步在腾讯云/阿里云网页控制台操作，不是命令）
```

```bash
# 22. 拿公网 IP（meta-staff .env 里填这个 IP）
curl -sS ifconfig.me
```

## 日常运维

```bash
cd ~/hermes-agent

docker compose logs -f gateway      # 实时日志
docker compose restart gateway      # 重启
docker compose down                 # 停
docker compose up -d                # 起
docker compose ps                   # 看状态

# 更新到最新版本
git pull && docker compose pull && docker compose up -d
```
